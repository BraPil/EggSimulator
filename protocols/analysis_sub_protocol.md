# Analysis Sub-Protocol

**Purpose**: Ensure thorough, comprehensive analysis of all results and information.
**Last Updated**: 2025-11-28

---

## Core Principle

Analyze every single word of every single result thoroughly and in full 100% comprehensiveness.

---

## Analysis Requirements

### 1. Completeness

- Read and process ALL content, not samples
- Do not skip sections regardless of perceived relevance
- Maintain awareness of the full context

### 2. Depth

- Identify explicit information
- Infer implicit relationships
- Note dependencies and connections
- Flag uncertainties or ambiguities

### 3. Documentation

- Record key findings
- Note analysis methodology used
- Document assumptions made
- List questions that arose

### 4. Verification

- Cross-reference findings with other sources when possible
- Validate conclusions against the prime directive
- Ensure analysis supports project goals

---

## When to Invoke

- Every time you receive information to process
- When reviewing code or documentation
- When evaluating options or making decisions
- When summarizing or reporting findings

---

## Output Format

Analysis results should include:

1. **Summary**: Brief overview of what was analyzed
2. **Key Findings**: Important discoveries or insights
3. **Relationships**: Dependencies, connections, patterns
4. **Recommendations**: Suggested actions based on analysis
5. **Open Questions**: Items requiring further investigation
